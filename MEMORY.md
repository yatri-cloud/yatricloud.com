## Project context
- **Brand name:** **Yatri Cloud** (this is the product name to use everywhere). The repo folder `yatri-practice-hub` is incidental — "practice hub" is NOT the brand; ignore it.
- **Product:** Yatri Cloud — Master IT Certifications. Cloud-cert platform (AWS/Azure/GCP/DevOps/K8s/Terraform). 50K+ learners, 4.8★.

## Initiative (started 2026-07-02): Complete UI redesign — DESIGN ONLY

- **Non-negotiable constraint:** change look/feel/motion only. **No** changes to routing, auth, API, data fetching, state, props, or business logic.

- **D35 — RFC 10008 HTTP QUERY METHOD (2026-07-03, commits 66fa358 + e369aa9).** RFC 10008 ("The HTTP QUERY Method", Proposed Standard June 2026 — post-cutoff, VERIFIED via WebFetch before implementing) = safe/idempotent/cacheable GET-with-body. Applied where semantics genuinely fit: admin gateway read actions (`invoices.list`/`payments.list`) accept QUERY (keeps token+params out of URL logs per the RFC's security guidance); mutations REJECTED over QUERY w/ 405 pre-auth; `Accept-Query: "application/json"` advertised; 415 on wrong content type. Client sends QUERY for reads with a **learned session-sticky POST fallback** (fetch error/405/501/415/400-missing-action → mark broken, retry POST) so behavior can never regress. NOT applied to supabase-js (library-controlled) or any mutation endpoint (semantically wrong). **PROVEN E2E in prod**: OPTIONS 204 w/ accept-query+allow headers; QUERY w/ fake token → OUR 401 from both curl AND browser fetch = Vercel routes QUERY + parses its body. ⚠️ Repeated fake-token API hits trip Vercel's security checkpoint (403 challenge HTML) — that's the WAF, not the method; space test requests out. Doc: docs/features/http-query-method.md. Honest note: benefit is semantics/security/future-cache, not measurable latency — the real perf lever is the ~3MB bundle (unsplit).
