# MEMORY.md — Yatri Cloud (long-term project memory)

> Durable decisions, context, and constraints for the UI redesign. Append; don't rewrite history.

## Project context
- **Brand name:** **Yatri Cloud** (this is the product name to use everywhere). The repo folder `yatri-practice-hub` is incidental — "practice hub" is NOT the brand; ignore it.
- **Product:** Yatri Cloud — Master IT Certifications. Cloud-cert platform (AWS/Azure/GCP/DevOps/K8s/Terraform). 50K+ learners, 4.8★.
- **Surfaces:** marketing (Index, Events, Store, Udemy), learning (Training, dashboards, Exam Dumps, Guides), community (Reviews, Achievements, Certified Yatris, events flows), trainer portal, admin dashboard. 32+ pages / 60+ routes.
- **Stack:** Vite + React + TS, React Router v6, Tailwind, shadcn/ui + Radix (49 primitives), Framer Motion, TanStack Query, Express `server.js`, Vercel. Font **Inter Tight**. Custom `ThemeProvider` (dark default).

## Initiative (started 2026-07-02): Complete UI redesign — DESIGN ONLY
- Re-skin the **entire** app to Lovable-grade craft, keeping the **blue** identity + true **light/dark**.
- **Non-negotiable constraint:** change look/feel/motion only. **No** changes to routing, auth, API, data fetching, state, props, or business logic.

## Design decisions (chronological)
- **D1 — Keep blue, drop cream.** Borrow Lovable's *craft* (opacity-driven neutrals, tactile inset depth, border-first containment, editorial spacing, tight display type). Do **not** adopt its cream palette or Camera Plain font. Blue stays `hsl(210 100% 50%)` / `#007CFF`.
- **D2 — Preserve token names.** All existing `src/index.css` variables keep their names so 49 shadcn primitives re-theme globally. New tokens (blue scale, shadows, motion, bands, status) are additive.
- **D3 — Palette FROZEN.** Keep existing blue (logo color) + entire palette unchanged; no repainting, no second accent. Redesign = layout/type/spacing/depth/structure/motion only. Contrast tweaks are optional/OFF-by-default and need approval (DESIGN.md §2.6).
- **D4 — "Light/dark" is dual-meaning.** (a) the theme toggle, AND (b) **alternating section background bands** within one theme for editorial rhythm, à la Lovable. Allowed backgrounds: **white, near-black, light-blue tint, and the logo blue itself** (blue is the only hue allowed as a bg wash; used like white/black). Blue feature band = white text, ≤1/page. Max 2–3 band switches/page; reserve inversion/blue for hero/proof/CTA. Education-platform psychology drives placement: trust first, calm focus for reading, motivation on CTAs/progress, quiet base-band on work surfaces (dashboards/admin).
- **D5 — Motion is a deliverable.** Framer Motion entrances/state/hover; awwwards/motionsites energy; transform/opacity only; always `prefers-reduced-motion` guarded.
- **D6 — Doc structure.** Adopted Nick Babich's Claude-Code design-project layout (CLAUDE.md, DESIGN.md, MEMORY.md, docs/, reference/, .claude/skills).

- **D7 — Build order:** design-system tokens first, then full homepage as visual proof-of-concept. (User choice, 2026-07-02.)
- **D8 — Default theme → LIGHT** (superseded by D11).
- **D11 — SINGLE THEME (no toggle).** User: "remove light and dark separate, keep only one default for everyone… white/black/blue, professional, not much saturated." `ThemeProvider` locked to `light` (removes `.dark`, clears stored theme; `useTheme` kept as no-op API). ThemeToggle removed from Navbar. Contrast via section bands only: white · black · blue · light-blue. `.dark` tokens kept but inactive.
- **D12 — Icons = Lucide (keep, don't mix); real tech logos added.** Official brand SVGs (AWS/Azure/GCP/K8s/Terraform/Docker/Ansible/Python/Linux/GitHub) in `public/logos/` via Devicon; reusable `TechLogos.tsx` `<LogoMarquee>` (white chips, grayscale→color, reduced-motion safe) wired into Curriculum marquee.
- **D9 — Display font added.** Keep Inter Tight for body/UI; add one distinctive **display face for headings** (chosen: Bricolage Grotesque, variable, editorial — swappable). Loaded via Fontsource/Google Fonts; body font unchanged.
- **D10 — Blue bands: Balanced.** Light-blue tint bands to group sections + exactly one saturated logo-blue feature band per page (white text) for the top CTA; canvas stays mostly white/black.

- **D13 — NO DUPLICATE UI + Section Creative Catalog.** Every section/page = distinct layout + signature motion; never the generic icon+title+description grid. Catalog of established homepage treatments lives in DESIGN.md §10 — consult before building to avoid repeating a spent pattern. Reuse shared *primitives* (bands, font-display, LogoMarquee, motion helpers), not layouts.
- **D14 — Creative dark finale.** Community (last section) + Footer unified as one designed dark closing: Community bottom gradients `#0a0a0a → #050505` with a labeled logo band; Footer = aurora blobs + dotted texture + glowing seam + giant shimmering "Yatri Cloud" wordmark + multi-column links. Removed the earlier 3-step journey ribbon per user.
- **D15 — Working agreement: update docs EVERY turn.** After each change update DESIGN.md (patterns/tokens/§10), root MEMORY.md (decision line), and persistent memory progress. New: `docs/SESSION-GUIDE.md` (how to behave in new sessions) + `docs/NEW-SESSION-PROMPT.md` (paste-ready kickoff prompt).

- **D16 — No black bands; mostly white + blue.** User: black bg under blue buttons looked bad. Converted Community + Footer from `#0a0a0a` dark → LIGHT. New band palette: white (most) · light-blue tint (Trust/Reviews/Community) · one saturated blue feature (Voucher). Footer = white with giant soft-blue "Yatri Cloud" wordmark. Also: Curriculum grid made uniform (bento span broke the last row before "View More"); CertFlow steps redesigned to numbered circle nodes on a straight animated connector (was a wavy squiggle); eligible exams show codes incl. fallback CLF-C02 / AIF-C01.

- **D17 — Brand voice + de-AI-ification.** New `docs/VOICE.md`: audience = **"Yatris"**, time-based `<YatriGreeting/>` ("Good morning, Yatris 👋", `src/components/YatriGreeting.tsx` + `.animate-wave`), warm/emotional/trustworthy copy (no generic AI filler), warm empty/loading states. **Removed ALL `Sparkles` icons** codebase-wide (→ contextual Lucide: FileSearch/PackageOpen/CalendarX/Award/Trophy/Wand2). Unified the old `components/Footer.tsx` → re-exports canonical `sections/Footer`. Redesigned pages this batch (voice+design+de-sparkle, design-only): ExamDumps, Events, EventDetail, YatriStore, Achievements, CertifiedYatris, certified-yatris/{AchievementsHeader,AchievementsSection}, Udemy, Training. Achievements PROVIDER_COLORS/BRAND_COLORS rainbow maps → unified on-palette blue. Greeting wired into homepage Hero. Competitor/psychology research in VOICE.md. **Remaining pages TODO** (dashboards, trainer/*, admin/*, event sub-forms, profile/legal/misc) — same rules.

- **D18 — Admin console redesign (design-only).** Entire `/admin` re-skinned to match the site. `AdminLayout` = production sidebar: desktop **collapsible** icon-rail (≤76px) ↔ full 64-width, remembered in `localStorage` (`admin_sidebar_collapsed`); mobile **off-canvas slider** (hamburger + blurred scrim, auto-close on route change); **single** collapse toggle (in the top header only — removed the duplicate sidebar/rail toggles). Top header = **solid** bar (not glass) with a route-aware breadcrumb "Admin › {page}", a "View site" link, and a profile chip; admin `<main>` gets a subtle `from-brand-50/40` wash so it's not stark white. `StatsCard` re-skinned to an **editorial** tile (big number + faint large watermark icon + colored accent bar — NOT the "colored icon box" AI cliché). All 17+ admin files mapped off-palette → tokens (0 off-palette, 0 Sparkles).
- **D19 — Admin header bands + form UX.** Every admin page (AdminEvents + 10 others) has a **blue-tinted header band** (`rounded-3xl border-brand-100 bg-gradient-to-br from-primary/[0.08] via-brand-50/50 to-card` + glow + eyebrow + title; stat cards moved inside). **CreateEvent** (standalone `/createevent`, no sidebar) = a **sticky solid action toolbar** (back + compact title + Publish buttons, `sticky top-0 z-30`, distinct from page) over a tinted page-header band (eyebrow + title + step progress). Admin **forms** (CreateEvent, TrainingManager wizard, Add Product, Add/Edit Exam Dump) got **numbered section headers** (badge `1/2/3…` + title + one-line helper + divider) so sections read like daily-task steps. AdminEvents tabs → segmented control with live counts.
- **D20 — Interaction/hover fixes (recurring class of bug).** shadcn `ghost`/`outline` Button variants force **white** hover text (`hover:text-accent-foreground`); when an agent adds a light `hover:bg-brand-50`/`hover:bg-destructive/10` without a text override you get **invisible white-on-light text**. FIX: always pair a light hover bg with `hover:text-primary` (blue controls) or make destructive/**remove** buttons **solid on hover** `hover:bg-destructive hover:text-destructive-foreground` (dark red + white — user wanted delete hovers clearly visible). Applied across all admin delete/remove + CreateEvent header buttons.
- **D21 — Greeting = home page ONLY.** `<YatriGreeting/>` lives only in the homepage `HeroSection`. Removed from all other pages (Events, EventDetail, ExamDumps, YatriStore, CertifiedYatris, Achievements, AchievementsSection; Training/Udemy kept their eyebrow pill with a static label). Also: removed the Hero "Women's Day offer · 50% OFF vouchers" pill (user, repeatedly); removed the ★ from the Hero "4.8" rating stat; slowed the footer "Yatri Cloud" wordmark shimmer to a relaxing `[animation-duration:9s] ease-in-out`.
